模型参数
使⽤提示词时，通常会通过 API 或直接与⼤语⾔模型进⾏交互。你可以通过配置⼀些参数以获得不同的提示结果，
在上⼀个章节中我们有提到过。调整这些设置对于提⾼响应的可靠性⾮常重要，你可能需要进⾏⼀些实验才能找出
适合您的⽤例的正确设置，如下所示：
1. Temperature：简单来说， temperature 的参数值越⼩，模型就会返回越确定的⼀个结果。如果调⾼该参
数值，⼤语⾔模型可能会返回更随机的结果，也就是说这可能会带来更多样化或更具创造性的产出。在实际
应⽤⽅⾯，对于质量保障（QA）等任务，我们可以设置更低的 temperature 值，以促使模型基于事实返回
更真实和简洁的结果。 对于诗歌⽣成或其他创造性任务，适度地调⾼ temperature 参数值可能会更好。
2. Top P：使⽤ top_p 也可以⽤来控制模型返回结果的确定性。如果你需要准确和事实的答案，就把参数值调
低。如果你在寻找更多样化的响应，可以将其值调⾼点。使⽤Top-P意味着只有词元集合（tokens）中包含
top_p 概率质量的才会被考虑⽤于响应，因此较低的 top_p 值会选择最有信⼼的响应。这意味着较⾼的
top_p 值将使模型考虑更多可能的词语，包括不太可能的词语，从⽽导致更多样化的输出。⼀般建议是改变

Temperature 和 Top-P 其中⼀个参数就⾏，不⽤两个都调整。
3. Max Tokens：您可以通过调整 max_tokens 来控制⼤模型⽣成的 token 数。指定 Max Tokens 有助于防⽌
⼤模型⽣成冗⻓或不相关的响应并控制成本。
4. Stop Sequence：模型会在⽣成过程中实时检查输出内容，如果遇到跟 stop 参数匹配的序列，就会中⽌⽣
成，即使 max_tokens 没⽤完。指定 stop 参数是控制⼤模型响应⻓度和结构的另⼀种⽅法。
5. Frequency Penalty： frequency_penalty 是对下⼀个⽣成的 token 进⾏惩罚，这个惩罚和 token 在响应
和提示中已出现的次数成⽐例， frequency_penalty 越⾼，某个词再次出现的可能性就越⼩，这个设置通
过给 重复数量多的 Token 设置更⾼的惩罚来减少响应中单词的重复。
6. Presence Penalty： presence_penalty 也是对重复的 token 施加惩罚，但与 frequency_penalty 不同
的是，惩罚对于所有重复 token 都是相同的。出现两次的 token 和出现 10 次的 token 会受到相同的惩罚。
此设置可防⽌模型在响应中过于频繁地⽣成重复的词。 如果您希望模型⽣成多样化或创造性的⽂本，您可以
设置更⾼的 presence_penalty ，如果您希望模型⽣成更专注的内容，您可以设置更低的
presence_penalty 。与 temperature 和 top_p ⼀样，⼀般建议是改变 frequency_penalty 和
presence_penalty 其中⼀个参数就⾏，不要同时调整两个。

思维树（ToT）
对于需要探索或预判战略的复杂任务来说，传统或简单的提示技巧是不够的。有⼈提出了思维树（ToT）框架，该
框架基于思维链提示进⾏了总结，引导语⾔模型探索把思维作为中间步骤来解决通⽤问题。ToT 维护着⼀棵思维
树，思维由连贯的语⾔序列表示，这个序列就是解决问题的中间步骤。使⽤这种⽅法，LLM 能够⾃⼰对严谨推理
过程的中间思维进⾏评估。LLM 将⽣成及评估思维的能⼒与搜索算法（如⼴度优先搜索和深度优先搜索）相结
合，在系统性探索思维的时候可以向前验证和回溯。
ToT 框架原理如下：

检索增强⽣成（RAG）
通⽤语⾔模型通过微调就可以完成⼏类常⻅任务，⽐如分析情绪和识别命名实体。这些任务不需要额外的背景知识
就可以完成。要完成更复杂和知识密集型的任务，可以基于语⾔模型构建⼀个系统，访问外部知识源来做到。这样
的实现与事实更加⼀性，⽣成的答案更可靠，还有助于缓解“幻觉”问题。
Meta AI 的研究⼈员引⼊了⼀种叫做检索增强⽣成（Retrieval Augmented Generation）的⽅法来完成这类知识密
集型的任务。RAG 把⼀个信息检索组件和⽂本⽣成模型结合在⼀起。RAG 可以微调，其内部知识的修改⽅式很⾼
效，不需要对整个模型进⾏重新训练。RAG 会接受输⼊并检索出⼀组相关/⽀撑的⽂档，并给出⽂档的来源（例如
维基百科）。这些⽂档作为上下⽂和输⼊的原始提示词组合，送给⽂本⽣成器得到最终的输出。这样 RAG 更加适
应事实会随时间变化的情况。这⾮常有⽤，因为 LLM 的参数化知识是静态的。RAG 让语⾔模型不⽤重新训练就能
够获取最新的信息，基于检索⽣成产⽣可靠的输出。这说明 RAG 是⼀种可⾏的⽅案，能在知识密集型任务中增强
语⾔模型的输出。最近，基于检索器的⽅法越来越流⾏，经常与 ChatGPT 等流⾏ LLM 结合使⽤来提⾼其能⼒和事
实⼀致性。

ReAct框架

ReAct 框架允许 LLM 与外部⼯具交互来获取额外信息，从⽽给出更可靠和实际的回应。结果表明，ReAct 可以在
语⾔和决策任务上的表现要⾼于⼏个最先进⽔准要求的的基线。ReAct 还提⾼了 LLM 的⼈类可解释性和可信度。
总的来说，作者发现了将 ReAct 和链式思考 (CoT) 结合使⽤的最好⽅法是在推理过程同时使⽤内部知识和获取到的
外部信息。
ReAct 的灵感来⾃于 “⾏为” 和 “推理” 之间的协同作⽤，正是这种协同作⽤使得⼈类能够学习新任务并做出决策或
推理。链式思考 (CoT) 提示显示了 LLMs 执⾏推理轨迹以⽣成涉及算术和常识推理的问题的答案的能⼒，以及其他
任务。但它因缺乏和外部世界的接触或⽆法更新⾃⼰的知识，⽽导致事实幻觉和错误传播等问题。
ReAct 是⼀个将推理和⾏为与 LLMs 相结合通⽤的范例。ReAct 提示 LLMs 为任务⽣成⼝头推理轨迹和操作。这使
得系统执⾏动态推理来创建、维护和调整操作计划，同时还⽀持与外部环境(例如，Wikipedia)的交互，以将额外信
息合并到推理中。下⾯是⼀个 ReAct 提示的示例(摘⾃论⽂，为简单起⻅缩短为⼀个示例:
