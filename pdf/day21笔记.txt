一、RAG介绍
1、什么是RAG？是检索增强生成
2、为什么需要RAG？
LLM的训练数据是有截止日期的，无法回答最新问题的
示例代码
from langchain_community.chat_models.tongyi import ChatTongyi
from langchain.chat_models import init_chat_model
from dotenv import load_dotenv

# llm = ChatTongyi(model='qwen3-max')
# result = llm.invoke('你最新的训练数据是哪一年多一天的？').content

print(result)

输出结果
我的知识截止日期是2025年5月。但具体到“哪一年多一天”这样的细节，由于训练数据涵盖多种来源和不同时间点的信息，无
法精确到“某一年比标准年份多一天”这种特殊日期。我的回答基于截至2025年5月的公开知识，如果需要最新动态，建议通过
联网搜索功能查询实时信息。

幻觉问题，生成一些答案看似合理其实是错误的（因为AI它的循环语料是日期的，你问的问题超出了它的限制，
这个时候又不好不给回答，因此会根据以往的经验给你一个答案）
缺乏专业知识，LLM可能不了解你的公司、品牌、内部的知识
以上存在的问题，可以通过RAG来解决
3、RAG解决方案
用户提问
检索相关知识（搭建本地知识库）
将检索出来的知识注入给提示词
生成精准的答案

三、TextSplitter 文档分隔
1、文档为什么要进行分隔？
问题：文档太长了，无法一次全部放进提示词
解决方案：进行分隔，可以把大文档切割成小块。例如：原始文档 10000字，切成4块，每块，就是2500字
2、分隔方法
split_document ，输入和输出类型都是一个列表。推荐，当咱们加载完文档后，直接使用这个方法，能保留文

件名和元数据
split_text ，输入类型是字符串，输出类型是一个列表，当咱们加载完文档后，只是字符，没有元数据

六、向量存储与检索
1、向量数据库
向量数据库能把文本（我喜欢素食）转成一组数字（向量），然后存储起来。当你输入一个问题时，它也会转成向
量，然后找出最相似的存储记录
2、工作原理
文本 --- 嵌入模型 --- 向量（一串数字）
向量存入数据库，同时保存原始文本
查询时，将查询文本转成向量，在数据库中距离最近的向量（计算，查看输入的问题和数据库里面的内容那个语
义是最相近的）
找到对应向量，然后把原始文本返回
3、向量数据库安装
ChromaDB 这个向量数据库是目前AI社区使用最多的一个，最大的特点就是对初学者极为友好，主打一个轻量级
和开箱即用
安装
pip install chromadb

from langchain_community.chat_models.tongyi import ChatTongyi
# RecursiveCharacterTextSplitter 分割器
from langchain_text_splitters import RecursiveCharacterTextSplitter
# DashScopeEmbeddings 阿里的文本嵌入模型（把文本进行向量化）
from langchain_community.embeddings import DashScopeEmbeddings
# Chroma 向量数据库
from langchain_community.vectorstores import Chroma
from dotenv import load_dotenv
from torch.ao.quantization.backend_config.onednn import embedding_op_dtype_configs
from torch.nn.utils import vector_to_parameters
load_dotenv()
llm = ChatTongyi(model='qwen3-max')
# 准备数据 --- 由于不是读取的数据，因此需要把文本到时转成标准文档格式进行处理
texts = [
"人工智能是研究如何让机器具有智能的学科",
"机器学习是人工智能的一个分支，让计算机从数据中学习",
"深度学习使用神经网络进行特征学习和预测",
"Python是一种广泛使用的高级编程语言",
"LangChain是一个用于构建LLM应用的框架"
]
# 创建文档分割器
# 行业标准做法：chunk_size=512，chunk_overlap=128 可以处理大部分的文档
splitter = RecursiveCharacterTextSplitter(
chunk_size=60,
chunk_overlap=10,
)
# create_documents(texts) 可以把文本转成标准文档格式再进行分隔
chunks = splitter.create_documents(texts)
# 创建嵌入模型
embeddings = DashScopeEmbeddings(model='text-embedding-v2')
# 创建向量数据库
# embedding 表示进行向量化处理
# documents 那个文档需要进行向量化
vectorstores = Chroma.from_documents(
embedding=embeddings,
documents=chunks,
persist_directory='./chroma_db'
)
# 查看向量数据库里面存入了多少个向量
# print(vectorstores._collection.count())
# 进行相似度检索

query = '什么是机器学习？'
# k=2 表示最终返回两个语义相似的结果
result = vectorstores.similarity_search(query, k=2)
for i, doc in enumerate(result):
print(f'{i}, {doc.page_content}')

八、RAG综合实战
from langchain_community.chat_models.tongyi import ChatTongyi
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_community.embeddings import DashScopeEmbeddings
from langchain_community.vectorstores import Chroma
from langchain_community.document_loaders import TextLoader
from langchain.agents import create_agent
from langchain_core.tools import tool
from langchain_community.vectorstores import Chroma
from dotenv import load_dotenv

load_dotenv()
llm = ChatTongyi(model='qwen3-max')
# 1、加载文档
loaders = TextLoader('./file/颐和园.txt', encoding='utf-8')
documents = loaders.load()
# 2、文档分隔
splitter = RecursiveCharacterTextSplitter(
chunk_size=512,
chunk_overlap=128,
)
chunks = splitter.split_documents(documents)
# 3、创建嵌入模型
embeddings = DashScopeEmbeddings(model='text-embedding-v2')
# 4、创建向量数据库
vectorstores = Chroma.from_documents(
embedding=embeddings,
documents=chunks,
persist_directory='./chroma_db2'
)
# 5、使用搜索器
retriever = vectorstores.as_retriever(search_kwargs={'k': 2})
# 6、创建工具
@tool
def yiheyuan_search(query: str):

""" 回答关于颐和园相关的历史、景点、建筑或相关的问题，使用此工具进行查询（搜索） """
# 调用检索器获取文档列表
docs = retriever.invoke(query)
# 把检索到的内容返回出去
return '\n\n'.join([d.page_content for d in docs])
# 7、创建智能体
agent = create_agent(
model=llm,
tools=[yiheyuan_search],
system_prompt='你是一个颐和园的智能导游助手。请根据用户的问题，利用提供的工具进行回答。如果工具中找不到
答案，请根据你的通用知识回答，或者诚实地说你不知道。'
)
result = agent.invoke({'messages': [('user', '颐和园的长廊有多长？')]})
print(result['messages'][-1].content)
