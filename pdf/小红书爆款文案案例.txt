三、数据持久化
1、问题：定义了一个数值类型的变量，初始值为0，当点击按钮后让其变量值每次+1，发送页面中实际上没有变化的
（只改变了一次）

import streamlit as st
a = 0
click_btn = st.button('加1')
if click_btn:
a += 1
st.write(a)
print(a)

2、原因：当修改代码后，streamlit每次都会自动更新，重新执行python文件，相当于是把代码从头到尾都重新运行
一遍，所以直接修改数据是不会变化的
3、解决方案：数据持久化
st.session_state 可以实现数据持久化

# {} 默认是一个空字典 --- st.session_state = {}
# print(st.session_state)
# 先查看a这个属性是否在st.session_state中存在
if 'a'not in st.session_state:
# 不存在
st.session_state['a'] = 0
click_btn = st.button('加1')
if click_btn:
st.session_state['a'] += 1
st.write(st.session_state['a'])

封装代码
import json
"""
params:
+ client OpenAI对象
+ system_prompt 表示的是系统提示词
+ few_shot_prompt 表示的是小样本提示词
+ user_prompt 表示的是用户提示词

+ model 表示的是模型名称
+ stream 表示是否开启流模型（打字效果，一个一个字输出）
+ * 表示它后面所有参数传递必须使用关键字参数
"""
def get_llm_response(client, *, system_prompt='', few_shot_prompt='', user_prompt='',
model='qwen3-max', stream=False):
# 定义聊天列表
messages = []
if system_prompt:
messages.append({'role': 'system', 'content': system_prompt})
if few_shot_prompt:
messages += json.loads(few_shot_prompt)
if user_prompt:
messages.append({'role': 'user', 'content': user_prompt})
# 补齐资源
resp = client.chat.completions.create(
model=model,
messages=messages,
stream=stream
)
# 判断是否开启了流模型
if not stream:
return resp.choices[0].message.content
return resp
