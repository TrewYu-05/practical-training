一、RAG介绍
1、什么是RAG？是检索增强生成
2、为什么需要RAG？
LLM的训练数据是有截止日期的，无法回答最新问题的
示例代码
from langchain_community.chat_models.tongyi import ChatTongyi
from langchain.chat_models import init_chat_model
from dotenv import load_dotenv
load_dotenv()
# llm = ChatTongyi(model='qwen3-max')
# result = llm.invoke('你最新的训练数据是哪一年多一天的？').content
# print(result)
llm = init_chat_model(model='deepseek-chat')
result = llm.invoke('你最新的训练数据是哪一年多一天的？').content
print(result)

输出结果
我的知识截止日期是2025年5月。但具体到“哪一年多一天”这样的细节，由于训练数据涵盖多种来源和不同时间点的信息，无
法精确到“某一年比标准年份多一天”这种特殊日期。我的回答基于截至2025年5月的公开知识，如果需要最新动态，建议通过
联网搜索功能查询实时信息。

幻觉问题，生成一些答案看似合理其实是错误的（因为AI它的循环语料是日期的，你问的问题超出了它的限制，
这个时候又不好不给回答，因此会根据以往的经验给你一个答案）
缺乏专业知识，LLM可能不了解你的公司、品牌、内部的知识
以上存在的问题，可以通过RAG来解决
3、RAG解决方案
用户提问
检索相关知识（搭建本地知识库）
将检索出来的知识注入给提示词
生成精准的答案

5、批量加载文件
from langchain_community.document_loaders import DirectoryLoader, TextLoader
from dotenv import load_dotenv
load_dotenv()
# 加载某个文件夹下的所有文件
# *.txt 表示匹配当前目录下所有的.txt文件 例如：a.txt

b.txt hello.txt

# **/ 表示递归匹配，意思是不管在那个文件夹下，都可以匹配到
# **/*.txt 整体的意思是可以匹配到./file文件夹下以及所以子文件夹里面的文件。文件后缀做了限定，只能匹配
到.txt文件
# loader_cls=TextLoader 使用那个模块加载文件
# show_progress=True 表示显示加载文件的进度
# loader_kwargs={'encoding': 'utf-8'} 设置字符编码
loader = DirectoryLoader(
path='./file',
glob='**/*.txt',
loader_cls=TextLoader,
show_progress=True,
loader_kwargs={'encoding': 'utf-8'}
)
documents = loader.load()
print(f'共加载了{len(documents)}个文件')
for document in documents:
print(document.page_content)
print('-' * 30)
print(document.metadata)

(KHTML, like Gecko) Chrome/147.0.0.0 Safari/537.36'}
